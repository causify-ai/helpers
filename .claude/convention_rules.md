# Important Guidelines and Rules

## Code Conventions for Different Types of Files

- For writing Python file (files with a `.py` extension) you MUST follow the
  rules from:
  - `.claude/skills/coding.rules.md`
  - `.claude/templates/code_template.py`

- For a file storing unit tests (files whose base name starts with
  `test/test_<file>.py`) you MUST follow the rules from:
  - `.claude/skills/testing.rules.md`
  - `.claude/templates/unit_test_template.py`

- For Jupyter notebook (files with `.ipynb` extension) you MUST follow the
  rules from
  - `.claude/skills/notebook.rules.md`
  - `.claude/templates/notebook_template.ipynb`
  - After modification you need to run `jupytext --sync`

- For writing markdown text you MUST follow instructions in
  `.claude/skills/markdown.rules.md`

- For writing README files you MUST follow instructions in
  `.claude/skills/readme.rules.md`

- For creating slides you MUST follow instructions in
  `.claude/skills/slides.rules.md`

- For writing LaTeX formulas and mathematical notation you MUST follow instructions in
  `.claude/skills/latex.rules.md`

- For writing blog posts you MUST follow instructions in
  `.claude/skills/blog.rules.md`

## Do Not Commit Code

- NEVER commit code to git without being explicitly asked to do it
