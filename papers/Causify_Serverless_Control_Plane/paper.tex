  \title{Serverless Control Plane, Container Data Plane:\\ \large Operating
  an Enterprise Causal AI Platform}

  % \author{\textbf{[Author Name]}\\
  % [Affiliation]\\
  % \texttt{[email]}}

  \date{\today}

  \begin{abstract}
    Enterprise AI platforms fail in a predictable way: compute scales, tenants
    multiply, and governance quietly gets smeared across everything that
    happens to be running. Authorization checks drift into services,
    tenant scoping leaks into query parameters, and the ``platform'' becomes
    a distributed guessing game about identity, tenancy, and intent.

    We describe a production architecture that makes the guessing illegal:
    a \emph{serverless control plane} that materializes tenant context into
    explicit, cacheable contracts, paired with a \emph{container data
    plane} that executes workloads without owning policy. Our control plane
    is an AWS Lambda API (Powertools) fronted by API Gateway. It
    authenticates users, derives effective context (including a safe
    \viewas{} mechanism), resolves reusable tenant scoping signals (e.g.,
    schema prefixes, tenant identifiers, user/session scope, connector keys,
    and job intent), minimizes API surface by filtering OpenAPI and
    endpoint access by enabled modules, and uses ETags and conditional requests
    to make ``who am I?'' fast and cheap at scale.

    The data plane runs long-lived services on Kubernetes (EKS) and can
    run burst compute as on-demand containers (e.g., ECS/Fargate). The key
    property is not the stack; it is the boundary: the control plane decides
    and attests; the data plane executes. We discuss design goals,
    contracts, middleware structure, defense-in-depth against confused-deputy
    and tenant-confusion risks, and operational outcomes that come from keeping
    governance thin, explicit, and boring.
  \end{abstract}
