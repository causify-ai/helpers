  \subsection{The core contract: \texttt{/v2/me}}
  The control plane's foundational endpoint is \texttt{/v2/me}. It returns:
  (i) immutable actor identity, (ii) effective context (possibly \viewas{}),
  and (iii) derived scoping signals required by downstream services.

  \newpage
  \begin{samepage}
    \begin{lstlisting}[style=json,caption={Representative \texttt{/v2/me} contract (simplified). Note the explicit actor vs effective context and derived \texttt{schema\_prefix}.},label={lst:v2me_cp}]
{
  "actor": {
    "email": "ada@acme.example",
    "tenant_id": "causify",
    "user_role": "admin",
    "groups": ["causify:admin", "acme:member"]
  },
  "view_as": {
    "view_as_self": false,
    "tenant_id": "acme",
    "user_role": "member",
    "schema_prefix": "acme"
  },
  "impersonatable": [
    { "tenant_id": "acme", "roles": ["demo", "member", "admin"] }
  ]
}
\end{lstlisting}
  \end{samepage}

  \paragraph{Caching semantics.}
  This endpoint is called frequently by the web tier. We therefore align
  it with HTTP semantics: responses include \texttt{ETag} and respect
  \texttt{If-None-Match} to return \texttt{304 Not Modified} when the
  effective contract has not changed \cite{rfc9110}. This makes "tell me
  who I am" a cheap question, which is important because the UI asks it
  like an anxious intern.

  Figure~\ref{fig:etag_flow} illustrates the two paths: a cache miss (full
  response with a fresh \etag{}) and a cache hit (\texttt{304 Not
  Modified} with no body).

  \subsection{Surface minimization: filtered OpenAPI and module gating}
  The control plane does not merely \emph{deny} unauthorized endpoints;
  it also avoids advertising them:
  \begin{itemize}
    \item \textbf{Filtered OpenAPI:} \texttt{/v1/openapi.json} is generated
      and then filtered so only endpoints corresponding to enabled modules
      are exposed in the spec.

    \item \textbf{Module-gated routers:} module endpoints are decorated
      with a guard (e.g., \texttt{@requires\_module("horizon")}) that checks
      effective enabled modules.
  \end{itemize}

  This reduces surface drift: clients and docs stop describing APIs that
  should not exist for the effective context.

% ==============================================================================
  \subsection{Downstream scoping: schema prefix injection}
  A common confused-deputy risk is tenant scoping provided by the client.
  We avoid that by deriving \texttt{schema\_prefix} (and similar scoping
  signals) in the control plane and injecting them into downstream calls.
  This ensures that the data plane does not need to infer tenant
  identity from user-provided parameters.
