\section{Limitations and Future Work}
\label{sec:limitations}

\paragraph{Single-organization evidence.}
All measurements come from one production deployment at Causify AI. The numbers are useful as operational evidence, not as a general benchmark.

\paragraph{AWS-specific implementation.}
The architecture uses API Gateway, Lambda, DynamoDB, Cognito, EKS, Fargate, RDS Proxy, and CloudWatch. Equivalent ideas exist on other clouds, but we have not validated the mapping.

\paragraph{Python-specific route enforcement.}
The deployed route decorator is Python-specific. Polyglot services need either a language-appropriate equivalent or enforcement moved closer to the gateway.

\paragraph{REST-first surface filtering.}
OpenAPI filtering works well for REST APIs. GraphQL and gRPC need separate strategies: schema pruning, resolver-level capability checks, or gateway-level method filtering.

\paragraph{Control-plane derivation remains load-bearing.}
The strongest remaining risk is wrong derivation inside the control plane. Downstream services can reject missing or malformed scope, but they cannot independently verify a wrong tenant prefix that the control plane derived consistently. Future work includes signed scope envelopes, stronger policy-versioning fields, and automated tenant-matrix tests that exercise representative actor/effective combinations.

\paragraph{Policy authoring still needs better tools.}
Raw JSON policy records are easy to store and hard to love. Future tooling should include JSON Schema validation, diffs, previews, staged rollout, and CI gates for tenant/module changes. Policy-as-data should not become policy-by-vibes.

\section*{Practitioner Takeaways}

\begin{itemize}
  \item \textbf{Materialize identity as a versioned contract.} Make actor, effective tenant, role, modules, impersonation, and scope visible in one response. Cache it, but treat caching as a correctness problem.
  \item \textbf{Derive scope once and inject it.} Let the control plane derive \scopeprefix{} from authenticated tenant state. Downstream services should reject missing scope, not invent it.
  \item \textbf{Filter the API surface early.} If a tenant cannot use a module, the generated API spec should not advertise it. Backend authorization still enforces the final decision.
  \item \textbf{Keep impersonation attributable.} Support engineers need customer perspective. They do not need shared accounts or log lines that erase the actor.
\end{itemize}
